Metoda SHAP¶

Opis danych¶

Dane pochodz z bazy o nazwie TUH Abnormal EEG Corpus (https://isip.piconepress.com/projects/tuh_eeg/downloads/tuh_eeg_abnormal/v2.0.0/). S to zapisy sygnau EEG z jednego szpitala sklasyfikowane jako prawidowe lub nieprawidowe.

Podstawowe informacje ułatwiające dalsze zrozumienie:¶

Do badania uyto standardowego ukadu elektrod 10-20, wyglda on nastpujco:

Do kadego sygnau EEG jest zaczony opis lekarza, ktry nie zawsze uywa konkretnych nazw elektrod, tylko bardziej oglne okrelenia.

  • Parzyste numery elektrod s po prawej stronie, nieparzyste - po lewej
  • Geneza literek w nazwach: pre-frontal (Fp), frontal (F), temporal (T), parietal (P), occipital (O), central (C)
  • Zwolnienia w obszarach mzgu, K kompleksy - niskie czstotliwoci
  • Vertex, sharp waves - wysze czstotliwoci
  • Wrzeciona snu (spindles) 12-16 Hz

Wybór cech¶

Lekarze oceniali sygnay EEG uywajc jedynie oczu i linijek - skupiali si wic na mocy danych czstotliwoci w danych kanaach. Aby mc porwnywa moje wyjanienia z ich opisami, wybraam te same cechy. Dla kadego kanau obliczyam moc sygnau w nastpujcych pasmach czstotliwoci:

  • 0-2 Hz (delta)
  • 1-3 Hz (delta)
  • 2-4 Hz (delta)
  • 3-6 Hz (theta)
  • 4-8 Hz (theta)
  • 6-10 Hz (alfa)
  • 8-13 Hz (alfa)
  • 10-15 Hz (beta)
  • 13-18 Hz (beta)
  • 15-21 Hz (beta)
  • 18-24 Hz (beta)
  • 21-27 Hz (beta)
  • 24-30 Hz (gamma)
  • 27-39 Hz (gamma)
  • 30-49 Hz (gamma) Uzyskaam w ten sposb 15x21=315 cech. Nie stosowaam adnych redukcji cech, poniewa chciaam uzyska peen obraz dla wszystkich cech.

W tych dwch przypadkach EEG rni si drastycznie. Dlatego te klasyfikator w tym przypadku potraktowa EEG podczas snu jako co nienormalnego. Zauway on wrzeciona snu dla elektrod T3, T4, T5, T6, zauway K-kompleksy w niych czstotliwociach oraz fale vertex w elektrodzie T6. Jest to cenna nauczka, aby w przyszoci trenowa osobno modele dla tych dwch przypadkw.

Podsumowanie istotności cech¶

Model duo bardziej preferuje praw pkul mzgu, Pz i Fz gwnie dla 10-15 Hz. Jest to ciekawy wynik, ktry wymaga dalszej analizy.

def TP(pred, true):
    return (pred == 1) * (true == 1)

def FP(pred, true):
    return (pred == 1) * (true == 0)

def TN(pred, true):
    return (pred == 0) * (true == 0)

def FN(pred, true):
    return (pred == 0) * (true == 1)
